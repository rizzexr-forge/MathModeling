# 📊 Симплекс-метод — Шпаргалка к зачёту

---

## 1. Что такое симплекс-метод?

**Симплекс-метод** — универсальный алгоритм решения задач линейного программирования с **любым числом переменных**, в отличие от графического метода (только 2 переменные).

Идея: двигаемся по **вершинам** области допустимых решений, улучшая значение целевой функции на каждом шаге.

---

## 2. Стандартная форма задачи

```
Максимизировать: Z = c₁x₁ + c₂x₂ + ... + cₙxₙ

При условиях:
  a₁₁x₁ + a₁₂x₂ + ... + a₁ₙxₙ  ≤  b₁
  a₂₁x₁ + a₂₂x₂ + ... + a₂ₙxₙ  ≤  b₂
  ...
  aₘ₁x₁ + aₘ₂x₂ + ... + aₘₙxₙ  ≤  bₘ
  x₁, x₂, ..., xₙ  ≥  0
```

> ⚠️ Все ограничения должны быть вида `≤`, правые части `bᵢ ≥ 0`.

---

## 3. Ключевые понятия

| Термин | Что это | В коде |
|---|---|---|
| **Slack-переменные (sᵢ)** | Добавляются к `≤`-ограничению, чтобы превратить его в равенство: `ax ≤ b` → `ax + s = b` | `slack_matrix = np.eye(n_constraints)` |
| **Каноническая форма** | Все ограничения — равенства; матрица содержит единичную подматрицу | `A_augmented = np.hstack([A, I])` |
| **Симплекс-таблица (tableau)** | Матрица, в которой хранятся коэффициенты всей системы + целевая функция | `self.tableau` |
| **Базисные переменные** | Переменные, которые «в базисе» — их значения ≠ 0 | `self.basic_vars` |
| **Небазисные переменные** | Переменные вне базиса — их значения = 0 | `self.non_basic_vars` |
| **Ведущий столбец** | Столбец переменной, вводимой в базис (с наим. отрицательным коэффициентом в строке Z) | `find_pivot_column()` |
| **Ведущая строка** | Строка переменной, выводимой из базиса (min отношение b/aᵢⱼ) | `find_pivot_row()` |
| **Ведущий элемент** | Элемент на пересечении ведущей строки и ведущего столбца | `tableau[pivot_row, pivot_col]` |
| **Поворот (pivot)** | Пересчёт всей таблицы — одна итерация симплекс-метода | `pivot()` |

---

## 4. Структура симплекс-таблицы

```
         x₁   x₂  ... xₙ  s₁  s₂  ... sₘ  | b
       ┌─────────────────────────────────────┐
s₁    │ a₁₁  a₁₂ ... a₁ₙ  1   0  ...  0  | b₁ │
s₂    │ a₂₁  a₂₂ ... a₂ₙ  0   1  ...  0  | b₂ │
...   │ ...                               ...  │
sₘ    │ aₘ₁  aₘ₂ ... aₘₙ  0   0  ...  1  | bₘ │
       ├─────────────────────────────────────┤
 Z    │ -c₁  -c₂ ... -cₙ  0   0  ...  0  | 0  │
       └─────────────────────────────────────┘
```

> В коде строка Z хранится как `-c` (чтобы признак оптимальности — все значения ≥ 0).

---

## 5. Алгоритм симплекс-метода (шаг за шагом)

```
Шаг 1. Привести к стандартной форме
        Добавить slack-переменные: aᵢx + sᵢ = bᵢ

Шаг 2. Построить начальную симплекс-таблицу
        tableau = [A | I | b]
        Строка Z = [-c | 0 | 0]

Шаг 3. Проверить оптимальность
        Если все коэффициенты строки Z ≥ 0 → СТОП, решение найдено

Шаг 4. Выбрать ведущий столбец (entering variable)
        pivot_col = argmin(строка Z)   ← самый отрицательный

Шаг 5. Выбрать ведущую строку (leaving variable)
        ratio = b[i] / tableau[i, pivot_col]  для всех i где tableau[i,j] > 0
        pivot_row = argmin(ratio)              ← правило минимального отношения

Шаг 6. Выполнить поворот (pivot)
        1) Делим ведущую строку на ведущий элемент
        2) Из всех остальных строк вычитаем нужный кратный ведущей строки,
           чтобы в ведущем столбце остался только 1 (в ведущей строке) и 0 (в остальных)

Шаг 7. Обновить список базисных переменных
        basic_vars[pivot_row] = pivot_col

Шаг 8. Перейти к шагу 3
```

## 6. Код — все методы с объяснением

### Инициализация
```python
solver = SimplexSolver(c, A, b, maximize=True)
# c — коэффициенты целевой функции, список длиной n
# A — матрица ограничений, размер m×n
# b — правые части, список длиной m
# maximize — True для max, False для min
```

### `create_initial_tableau()` — построение начальной таблицы
```python
# 1. Добавляем единичную матрицу (slack-переменные)
slack_matrix = np.eye(n_constraints)          # I размером m×m

# 2. Расширяем матрицу A → [A | I]
A_augmented = np.hstack([A, slack_matrix])

# 3. Для минимизации меняем знак: min cx ≡ max(-cx)
if not maximize:
    c_full = -c_full

# 4. Строим таблицу размером (m+1) × (n+m+1)
tableau[:-1, :-1] = A_augmented    # Верхняя часть: коэффициенты
tableau[:-1, -1]  = b              # Правый столбец: правые части
tableau[-1, :-1]  = -c_full        # Строка Z: отрицательные коэф-ты (признак для поиска)
```

### `find_pivot_column()` — ведущий столбец
```python
last_row = tableau[-1, :-1]      # Строка Z без последнего столбца
min_val = np.min(last_row)
if min_val >= 0:
    return -1                    # Нет отрицательных → решение оптимально!
pivot_col = np.argmin(last_row)  # Самый отрицательный → вводим в базис
```

### `find_pivot_row()` — ведущая строка (правило min отношения)
```python
for i in range(n_constraints):
    if tableau[i, pivot_col] > 0:
        ratio = tableau[i, -1] / tableau[i, pivot_col]  # b[i] / a[i,j]
        ratios.append(ratio)
    else:
        ratios.append(np.inf)   # Отрицательные и нулевые — не подходят

pivot_row = ratios.index(min(ratios))   # Строка с минимальным отношением
# Если все inf → задача неограничена!
```

### `pivot()` — поворот (пересчёт таблицы)
```python
# 1. Обновляем базис: pivot_col входит, basic_vars[pivot_row] выходит
basic_vars[pivot_row] = pivot_col

# 2. Нормализуем ведущую строку (делим на ведущий элемент)
pivot_element = tableau[pivot_row, pivot_col]
tableau[pivot_row, :] /= pivot_element       # Теперь ведущий элемент = 1

# 3. Обнуляем ведущий столбец во всех остальных строках
for i in range(n_constraints + 1):           # +1 потому что строка Z тоже
    if i != pivot_row:
        factor = tableau[i, pivot_col]
        tableau[i, :] -= factor * tableau[pivot_row, :]
# После этого: в столбце pivot_col — единица только в строке pivot_row
```

### `get_solution()` — извлечение решения
```python
solution = np.zeros(n_vars + n_constraints)

# Значения базисных переменных — берём из правого столбца
for i, var_idx in enumerate(basic_vars):
    solution[var_idx] = tableau[i, -1]

# Значение Z — в правом нижнем углу таблицы
objective_value = tableau[-1, -1]
if not maximize:
    objective_value = -objective_value   # Возвращаем знак обратно

return {
    'x':          solution[:n_vars],   # Основные переменные x₁...xₙ
    'slack':      solution[n_vars:],   # Slack-переменные s₁...sₘ
    'z':          objective_value,     # Оптимальное Z
    'basic_vars': basic_vars           # Индексы базисных переменных
}
```

## 7. Три тестовых примера из лабораторной

### Тест 1 — Максимизация (2 переменные)
```python
# max Z = 3x₁ + 5x₂
# x₁        ≤ 4
#     2x₂   ≤ 12
# 3x₁ + 2x₂ ≤ 18
# x₁, x₂ ≥ 0

c = [3, 5]
A = [[1, 0], [0, 2], [3, 2]]
b = [4, 12, 18]

solver = SimplexSolver(c, A, b, maximize=True)
solution = solver.solve()
# Ответ: x₁ = 2, x₂ = 6, Z = 36
```

### Тест 2 — Минимизация (ограничения типа ≥)
```python
# min Z = 6x₁ + 8x₂
# 3x₁ + x₂  ≥ 4   →  умножаем на -1 → -3x₁ - x₂  ≤ -4
#  x₁ + 2x₂ ≥ 4   →  умножаем на -1 →  -x₁ - 2x₂ ≤ -4

c = [6, 8]
A = [[-3, -1], [-1, -2]]   # Знаки перевёрнуты!
b = [-4, -4]                # Правые части тоже

solver = SimplexSolver(c, A, b, maximize=False)
solution = solver.solve()
# Ответ: x₁ = 0.8, x₂ = 1.6, Z = 17.6
```

> 🔑 **Важно:** ограничение `≥` преобразуется умножением обеих частей на `-1`, чтобы получить `≤`.

### Тест 3 — Максимизация (3 переменные)
```python
# max Z = 4x₁ + 6x₂ + 3x₃
# x₁ +  x₂ +  x₃ ≤ 100
# 10x₁ + 4x₂ + 5x₃ ≤ 600
# 2x₁ + 2x₂ + 6x₃ ≤ 300

c = [4, 6, 3]
A = [[1,1,1],[10,4,5],[2,2,6]]
b = [100, 600, 300]

solver = SimplexSolver(c, A, b, maximize=True)
solution = solver.solve()
```

## 8. Преобразование типов ограничений

| Исходный вид | Как преобразовать | Что добавить |
|---|---|---|
| `ax ≤ b` | Стандартная форма | Slack: `ax + s = b` (s ≥ 0) |
| `ax ≥ b` | Умножить на -1 → `(-a)x ≤ -b` | Slack: `(-a)x + s = -b` |
| `ax = b` | Двухфазный метод (в данном коде не реализован) | Artificial variable |
| min `cx` | Заменить на max `(-c)x` | `c_full = -c_full` в коде |

---

## 9. Признаки оптимальности и особые случаи

| Ситуация | Признак | Что происходит |
|---|---|---|
| **Оптимум найден** | Все коэф-ты строки Z ≥ 0 | `find_pivot_column()` вернул `-1`, цикл стоп |
| **Задача неограничена** | В ведущем столбце нет положительных элементов | `find_pivot_row()` выбрасывает `ValueError("Задача неограничена")` |
| **Вырожденность** | Одно из отношений b/a = 0 | Следующая итерация не улучшает Z (цикл Bland'а) |
| **Альтернативные оптимумы** | Коэф-т небазисной переменной в строке Z = 0 | Несколько точек с одинаковым Z |

## 10. Сложность алгоритма

| Параметр | Значение |
|---|---|
| Размер таблицы | (m+1) × (n+m+1), где m — ограничения, n — переменные |
| Число итераций (средн.) | O(m) — практически быстрый |
| Число итераций (худш.) | O(2ⁿ) — экспоненциальный (теоретически) |
| Одна итерация | O(m·(n+m)) — пересчёт строк таблицы |

## 11. Ответы на контрольные вопросы

**Q: Как преобразовать задачу минимизации в максимизацию?**
> Умножить все коэффициенты целевой функции на -1: `min cx` = `max(-cx)`. В коде: `c_full = -c_full`.

**Q: Что такое базисные и небазисные переменные?**
> Базисные — те, у которых в симплекс-таблице есть единичный столбец; их значения ≠ 0 (обычно). Небазисные — равны 0.

**Q: Как определяется оптимальность текущего решения?**
> Если все коэффициенты в строке целевой функции (строка Z) неотрицательны — решение оптимально.

**Q: По какому критерию выбирается переменная для ввода в базис?**
> Выбирается столбец с **наименьшим (самым отрицательным)** коэффициентом в строке Z.

**Q: По какому критерию выбирается переменная, выводимая из базиса?**
> Правило минимального отношения: `min(bᵢ / aᵢⱼ)` при `aᵢⱼ > 0`. Гарантирует неотрицательность решения.

**Q: Что такое вырожденность?**
> Когда одна из базисных переменных равна 0. Возможны циклы без улучшения Z. Решение — правило Bland'а (выбирать переменную с минимальным индексом).

**Q: Геометрическая интерпретация?**
> Каждая итерация — переход из одной вершины многогранника ОДР в соседнюю с лучшим значением Z.

**Q: Преимущества и недостатки?**
> ✅ Работает с любым числом переменных, эффективен на практике.
> ❌ Экспоненциальная сложность в худшем случае, не работает напрямую с ограничениями `≥` и `=` без преобразований.

## 12. Быстрый запуск — минимальный код

```python
import numpy as np

# 1. Создать и решить задачу
solver = SimplexSolver(
    c=[3, 5],                      # Целевая функция
    A=[[1,0],[0,2],[3,2]],         # Матрица ограничений
    b=[4, 12, 18],                 # Правые части
    maximize=True                  # Максимизация
)

solution = solver.solve()          # Запуск симплекс-метода
solver.print_solution(solution)    # Вывод результата

# Получить конкретные значения:
print(solution['x'])    # Массив [x₁, x₂, ...]
print(solution['z'])    # Значение Z
print(solution['slack'])# Slack-переменные
```

## 13. Формулы — итоговая таблица

| Формула | Назначение |
|---|---|
| `Z = c₁x₁ + ... + cₙxₙ` | Целевая функция |
| `aᵢx + sᵢ = bᵢ` | Канонизация ограничения `≤` через slack |
| `pivot_col = argmin(строка Z)` | Выбор ведущего столбца |
| `ratio = bᵢ / aᵢⱼ` (при `aᵢⱼ > 0`) | Вычисление отношений для ведущей строки |
| `pivot_row = argmin(ratio)` | Выбор ведущей строки |
| `row_pivot /= a_pivot` | Нормализация ведущей строки |
| `rowᵢ -= factor · row_pivot` | Обнуление остальных строк в ведущем столбце |
| `Z* = tableau[-1, -1]` | Оптимальное значение после окончания |

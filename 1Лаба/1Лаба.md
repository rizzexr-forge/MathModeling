# 📈 Графический метод ЛП — Шпаргалка к зачёту

---

## 1. Что такое задача линейного программирования (ЛП)?

**Задача ЛП** — найти значения переменных `x₁, x₂`, при которых целевая функция достигает максимума или минимума, при соблюдении набора линейных ограничений.

### Стандартная форма
```
Z = c₁·x₁ + c₂·x₂  →  max (или min)

при ограничениях:
  aᵢ·x₁ + bᵢ·x₂  ≤  cᵢ  (или ≥)
  x₁ ≥ 0,  x₂ ≥ 0
```

---

## 2. Ключевые понятия

| Термин | Что это | В коде |
|---|---|---|
| **Целевая функция** | `Z = c₁x₁ + c₂x₂`, которую max/min-изируем | `problem.evaluate_objective(x1, x2)` |
| **Ограничение** | Неравенство вида `ax₁ + bx₂ ≤ c` | Класс `Constraint(a, b, c, '<=')` |
| **ОДР** (Область Допустимых Решений) | Многоугольник, где выполняются **все** ограничения | `find_feasible_region_vertices()` |
| **Вершины ОДР** | Угловые точки многоугольника ОДР | Список `vertices` |
| **Оптимальная точка** | Вершина ОДР, где Z → max/min | `find_optimal_solution()` |
| **Линия уровня** | Линия `Z = const`, параллельный сдвиг к оптимуму | Метод `_plot_objective_lines()` |

---

## 3. Алгоритм графического метода (шаги)

```
1. Построить ОДР — нарисовать все ограничения как прямые,
   закрасить пересечение допустимых полуплоскостей.

2. Найти все вершины ОДР — точки пересечения граничных прямых,
   удовлетворяющие всем ограничениям.

3. Вычислить Z в каждой вершине: Z = c₁·x₁ + c₂·x₂

4. Выбрать оптимальную вершину:
   - для max: вершина с наибольшим Z
   - для min: вершина с наименьшим Z
```

## 4. Классы и их роль

### `Constraint` — одно ограничение
```python
class Constraint:
    def __init__(self, a, b, c, constraint_type='<='):
        # a·x₁ + b·x₂  <=/>= c
        self.a = a
        self.b = b
        self.c = c
        self.type = constraint_type   # '<=' или '>='

    def evaluate(self, x1, x2):
        # Возвращает левую часть: a·x₁ + b·x₂
        return self.a * x1 + self.b * x2

    def is_satisfied(self, x1, x2, tol=1e-6):
        # True если точка (x1, x2) удовлетворяет ограничению
        value = self.evaluate(x1, x2)
        if self.type == '<=':
            return value <= self.c + tol
        else:
            return value >= self.c - tol
```

### `LinearProgrammingProblem` — вся задача ЛП
```python
problem = LinearProgrammingProblem(c1=3, c2=2, objective='max')
# Автоматически добавляет x₁ >= 0, x₂ >= 0

problem.add_constraint(Constraint(2, 1, 100, '<='))  # 2x₁ + x₂ ≤ 100
problem.add_constraint(Constraint(1, 1, 80,  '<='))  # x₁  + x₂ ≤ 80
problem.add_constraint(Constraint(1, 0, 40,  '<='))  # x₁      ≤ 40

result = problem.find_optimal_solution()
# result['optimal_point'] = (x1*, x2*)
# result['optimal_value'] = Z*
# result['vertices']      = [(x1, x2), ...]
# result['status']        = 'optimal' / 'infeasible'
```

### `GraphicalOptimizer` — визуализация
```python
optimizer = GraphicalOptimizer(problem)
fig, ax = optimizer.plot_solution(figsize=(10, 8))
optimizer.save_plot('graphical_solution.png')
```

### `SensitivityAnalyzer` — анализ чувствительности
```python
analyzer = SensitivityAnalyzer(problem)

```

## 5. Как находятся вершины ОДР

### Шаг 1: Пересечение каждой пары ограничений
```
aᵢx₁ + bᵢx₂ = cᵢ
aⱼx₁ + bⱼx₂ = cⱼ
```
Решается как система 2×2:
```
det = aᵢ·bⱼ - aⱼ·bᵢ

x₁ = (cᵢ·bⱼ - cⱼ·bᵢ) / det
x₂ = (aᵢ·cⱼ - aⱼ·cᵢ) / det
```
В коде: `_intersect_constraints(c1, c2)`

### Шаг 2: Проверка допустимости точки
```python
def is_feasible_point(self, x1, x2):
    for constraint in self.constraints:
        if not constraint.is_satisfied(x1, x2):
            return False
    return True
```
Оставляем только точки, которые удовлетворяют **всем** ограничениям.

## 6. Особые случаи

| Случай | Признак | Статус |
|---|---|---|
| **Нормальный** | ОДР — выпуклый многоугольник, Z ограничена | `'optimal'` |
| **Недопустимая задача** | ОДР пуста (нет точек, удовлетворяющих всем ограничениям) | `'infeasible'` |
| **Неограниченное решение** | ОДР уходит в бесконечность в направлении роста Z | Не найдено max |
| **Альтернативные оптимумы** | Целевая функция параллельна стороне ОДР — оптимум на целом отрезке | Два угла с равным Z |

## 7. Анализ чувствительности

**Что это:** изучение того, как меняется оптимальное решение при изменении коэффициентов.

### Активное ограничение (дефицитное)
- Ресурс использован **полностью**: `a·x₁* + b·x₂* = c`
- Запас = 0 → ограничение "связывает" оптимум

### Неактивное ограничение
- Ресурс использован **не полностью**: `a·x₁* + b·x₂* < c`
- Запас > 0 → ограничение не влияет на оптимум

```python
# Вычисление запаса для ограничения типа <=:
slack = constraint.c - constraint.evaluate(x1_opt, x2_opt)
# slack == 0  → активное (дефицитное)
# slack > 0   → неактивное (есть резерв)
```

### Тепловые карты
Показывают, как меняется x₁*, x₂*, Z* при изменении c₁ и c₂:
```python
analyzer = SensitivityAnalyzer(problem)
fig = analyzer.plot_sensitivity_heatmap()
# 4 графика: x₁(c₁,c₂), x₂(c₁,c₂), Z(c₁,c₂), Z(c₁) при фикс. c₂
```

## 8. Проверка через scipy

```python
from scipy.optimize import linprog

# linprog ВСЕГДА минимизирует, поэтому для max: умножаем на -1
c = [-3, -2]          # max 3x₁ + 2x₂  ≡  min -3x₁ - 2x₂

A_ub = [[2, 1],       # 2x₁ + x₂ ≤ 100
        [1, 1],       # x₁  + x₂ ≤ 80
        [1, 0]]       # x₁       ≤ 40
b_ub = [100, 80, 40]

bounds = [(0, None), (0, None)]  # x₁ ≥ 0, x₂ ≥ 0

result = linprog(c, A_ub=A_ub, b_ub=b_ub, bounds=bounds, method='highs')

x1_opt, x2_opt = result.x
Z_max = -result.fun   # обратный знак!
```

> ⚠️ `linprog` использует `method='highs'` — это современный симплекс-метод, более точный и быстрый.

## 10. Быстрый пример — запуск с нуля

```python
from linear_programming import LinearProgrammingProblem, Constraint, GraphicalOptimizer

# 1. Создать задачу
problem = LinearProgrammingProblem(c1=3, c2=2, objective='max')

# 2. Добавить ограничения
problem.add_constraint(Constraint(2, 1, 100, '<='))
```

## 11. Формулы — итоговая таблица

| Формула | Назначение |
|---|---|
| `Z = c₁x₁ + c₂x₂` | Целевая функция |
| `aᵢx₁ + bᵢx₂ ≤ cᵢ` | Ограничение типа "≤" |
| `det = aᵢbⱼ - aⱼbᵢ` | Определитель системы (≠0 → пересечение есть) |
| `x₁ = (cᵢbⱼ - cⱼbᵢ) / det` | x₁ точки пересечения двух прямых |
| `x₂ = (aᵢcⱼ - aⱼcᵢ) / det` | x₂ точки пересечения двух прямых |
| `slack = c - (ax₁* + bx₂*)` | Запас ресурса (0 = активное ограничение) |
| `Z* = c₁x₁* + c₂x₂*` | Оптимальное значение целевой функции |

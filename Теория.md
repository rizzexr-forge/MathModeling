**1\. Математические методы и моделирование** - это подходы к изучению объектов через их математические модели (совокупность уравнений, неравенств, логических условий). Методы включают аналитические, численные, имитационные. Моделирование позволяет прогнозировать поведение системы без натурных экспериментов.

**2\. Вычислительный эксперимент и компьютерное моделирование** - это исследование модели с помощью программных средств. Компьютер многократно решает уравнения при разных параметрах, что позволяет анализировать сценарии, оптимизировать режимы, визуализировать результаты там, где аналитическое решение невозможно или трудоёмко.

**3\. Линейное программирование (ЛП)** - раздел оптимизации, где целевая функция и ограничения линейны. Задачи ЛП решаются симплекс-методом, графически (для двух переменных) или с помощью двойственности. Применяется в планировании, логистике, экономике.

**4\. Постановка задачи ЛП** - найти максимум или минимум линейной функции \\( c^T x \\) при ограничениях \\( Ax \\le b \\) (или \\( =, \\ge \\)) и \\( x \\ge 0 \\). Требуются: переменные, целевая функция, система линейных ограничений, условие неотрицательности.

**5\. Виды записи задач ЛП:** общая (смешанные неравенства и равенства), стандартная (максимизация, ограничения типа ≤, \\( x \\ge 0 \\)), каноническая (минимизация, равенства, \\( x \\ge 0 \\)). Любую форму можно свести к канонической добавлением балансовых переменных.

**6\. Повторение** - см. п. 4 и 5. Дополнительно: матричная форма \\( \\max c^T x,\\ Ax \\le b,\\ x \\ge 0 \\) и векторная.

**7\. Графическое решение задачи ЛП** - для двух переменных строятся прямые ограничений, находится область допустимых решений (ОДР). Целевая функция задаёт линии уровня. Оптимум достигается в вершине ОДР. Перебором вершин находим экстремум.

**8\. Графический способ оптимизации** - аналогичен п. 7. Позволяет наглядно определить наличие решения, его единственность или альтернативный оптимум (если линия уровня параллельна ребру), а также неограниченность задачи.

**9\. Симплексный метод** - итерационный алгоритм решения ЛП. Начинает с угловой точки (базисного решения), переходит к соседней вершине с улучшением целевой функции. Критерий оптимальности - неотрицательность симплекс-разностей (для максимума). Заканчивается в оптимуме или фиксирует неограниченность.

**10\. Автоматизация симплекс-метода** - реализация в виде компьютерных программ (Excel, MATLAB, Python). Используются симплекс-таблицы или матричные вычисления. Автоматически выбирается разрешающий столбец/строка, выполняются преобразования, проверяются условия остановки.

**11\. Двойственные задачи** - каждой задаче ЛП (прямой) соответствует двойственная. Переменные двойственной интерпретируются как оценки ресурсов. Связь: если прямая на максимум с ограничениями ≤, то двойственная на минимум с ≥. Выполняется теорема двойственности.

**12\. Решение двойственных задач** - можно решить симплекс-методом, либо получить из последней симплекс-таблицы прямой задачи (оценки - теневые цены). По теореме равновесия: в оптимуме значения целевых функций равны, а переменные и ограничения связаны условиями дополняющей нежёсткости.

**13\. Транспортные задачи** - частный случай ЛП о перевозке однородного груза от поставщиков к потребителям с минимизацией стоимости. Исходные данные: запасы, потребности, матрица тарифов. Условие баланса - сумма запасов равна сумме потребностей (открытая/закрытая).

**14\. Решение транспортных задач методом потенциалов** - после построения начального опорного плана (методом северо-западного угла или минимального элемента) вычисляются потенциалы строк (u_i) и столбцов (v_j). Для свободных клеток считаются оценки (u_i+v_j - c_ij). Если все неотрицательны - план оптимален, иначе цикл пересчёта.

**15\. Метод Фогеля** - эвристика для начального опорного плана в транспортной задаче. Для каждой строки/столбца вычисляется штраф (разность между двумя минимальными тарифами). Выбирается максимальный штраф, в клетку с минимальным тарифом помещается максимально возможная поставка. Уменьшает количество итераций.

**16\. Метод северо-западного угла** - простейший способ начального распределения в транспортной задаче. Заполнение начинается с левой верхней клетки. Берётся минимум от остатка поставщика и потребности, остатки корректируются, движение идёт вправо или вниз. План далёк от оптимального, но легко строится.

**17\. Графовые модели** - представление систем в виде графа (вершины - объекты, рёбра/дуги - связи). Применяются для моделирования сетей, маршрутов, зависимостей. Решаются задачи поиска путей, потоков, остовов, раскраски и т.д.

**18\. Общие сведения о теории графов** - граф G(V,E), где V - вершины, E - рёбра. Ориентированные и неориентированные, взвешенные и невзвешенные. Основные понятия: степень вершины, путь, цикл, связность, матрицы смежности и инцидентности.

**19\. Дерево как частный случай представления** графа - связный ациклический граф. У него рёбер на 1 меньше, чем вершин. Любые две вершины соединены единственным путём. Используется в иерархиях, алгоритмах поиска, построении минимальных остовов.

**20\. Остовное дерево (каркас)** - подграф связного графа, являющийся деревом и включающий все вершины исходного графа. Для взвешенных графов ищут минимальное остовное дерево (MST) - сумма весов минимальна.

**21\. Алгоритмы Прима и Краскала** - оба для нахождения MST. Прим: начинается с одной вершины, добавляется ближайшее ребро к растущему дереву. Краскал: сортирует рёбра по весу и добавляет, если не образует цикл. Сложность: O(E log V).

**22\. Кратчайшие пути в графе** - задача найти путь минимальной суммарной длины (веса) между двумя вершинами. Для неотрицательных весов - алгоритм Дейкстры, для любых (без отрицательных циклов) - Флойда-Уоршелла или Беллмана-Форда.

**23\. Алгоритм Дейкстры** - находит кратчайшие пути от одной вершины до всех остальных в графе с неотрицательными весами. Поддерживает множество непосещённых вершин, на каждом шаге выбирает вершину с минимальным текущим расстоянием и релаксирует рёбра.

**24\. Нахождение кратчайших путей между точками с использованием алгоритма Флойда** - метод динамического программирования для всех пар вершин. Итеративно улучшает оценку расстояния: d\[i\]\[j\] = min(d\[i\]\[j\], d\[i\]\[k\]+d\[k\]\[j\]). Работает с отрицательными рёбрами без отрицательных циклов. Сложность O(n^3).

**25\. Нахождение кратчайших путей в графе** - общее название. В зависимости от графа: Дейкстра (одна вершина, неотрицательные), Беллман-Форд (есть отрицательные), Флойд (все пары), A\* (эвристика). Применяется в навигации, сетях.

**26\. Поиск наикратчайшего расстояния между вершинами графа** - частный случай п.25. Можно использовать Дейкстру для одной пары (с остановкой при достижении цели) или BFS для единичных весов.

**27\. Дискретное программирование** - оптимизация, где часть или все переменные принимают дискретные (целые, булевы) значения. Примеры: целочисленное ЛП, задачи о рюкзаке, коммивояжёре. Решается методами ветвей и границ, отсечений (Гомори).

**28\. Метод ветвей и границ** - универсальный метод для дискретной оптимизации. Задача разбивается на подзадачи (ветвление), для каждой вычисляется оценка (граница) оптимума. Ветви, где оценка хуже текущего рекорда, отсекаются. Гарантирует глобальный оптимум.

**29\. Метод Гомори** - для целочисленного ЛП. Сначала решается непрерывная задача симплексом. Если решение нецелое, добавляется дополнительное ограничение (отсечение Гомори), отсекающее нецелую вершину, но не отсекающее целые точки. Повторяется до целочисленности.

**30\. Решение задач по методу Гомори** - после каждого симплекс-шага проверяется целочисленность базисных переменных. Для дробной переменной строится отсечение на основе строки симплекс-таблицы. Новая строка добавляется, задача решается заново. Процесс конечен.

**31\. Задачи целочисленного линейного программирования** - ЛП с требованием целочисленности всех или некоторых переменных. Сложнее непрерывных (NP-трудные). Примеры: распределение оборудования, планирование производства неделимыми партиями.

**32\. Решение задач по методу ветвей и границ** - на примере целочисленного ЛП: решается непрерывный аналог. Если переменная x_i = 2.3, создаются две ветви: x_i ≤ 2 и x_i ≥ 3. Для каждой ветви решается непрерывная задача. Процесс рекурсивно продолжается.

**33\. Динамическое программирование (ДП)** - метод решения задач, обладающих свойством аддитивности и отсутствия последействия. Оптимальное решение строится по этапам: от начального состояния к конечному, с использованием рекуррентных соотношений.

**34\. Общие принципы решения задач методом ДП** - разбиение на этапы, определение состояний на каждом этапе, рекуррентное соотношение перехода, вычисление от конца к началу или наоборот. Хранятся оптимальные значения для каждого состояния (принцип Беллмана).

**35\. Принцип оптимальности Беллмана. Уравнения Беллмана** - «остаток оптимального пути сам является оптимальным». Уравнение: \\( f*k(s) = \\min*{a} \[ g*k(s,a) + f*{k+1}(s') \] \\), где f_k - оптимальное значение с этапа k из состояния s, a - решение.

**36\. Решение производственных задач методом ДП** - например, распределение ресурсов по периодам или инвестиций по проектам. Определяются этапы (годы, предприятия), состояние (оставшийся ресурс), управление (сколько вложить). Целевая функция - максимум прибыли.

**37\. Сетевые модели задач динамического программирования** - задачи представляются как ориентированный ациклический граф, где вершины - состояния, дуги - переходы с выигрышем. Нахождение оптимального пути (максимум или минимум суммы) - классическое ДП (например, задача о замене оборудования, кратчайший путь).

**38\. Нахождение кратчайшего маршрута** - частный случай ДП на сети (см. п.24, 25). Для ациклического графа можно использовать ДП с топологической сортировкой. Для общего графа без отрицательных циклов - Беллман-Форд или Флойд.

**39\. Матричный способ задания сетей** - граф задаётся матрицей смежности (A\[i\]\[j\]=вес ребра или 0/∞), матрицей инцидентности (строки - вершины, столбцы - рёбра), матрицей расстояний. Удобно для алгоритмов Флойда, Дейкстры, умножения матриц.

**40\. Математические пакеты прикладных программ** - системы для численных и символьных вычислений, визуализации, моделирования. Примеры: MATLAB, Mathematica, Maple, SciPy/NumPy (Python). Содержат готовые функции для ЛП, графов, ДП, решения уравнений.

**41\. Общий обзор пакета MATLAB** - среда для матричных вычислений, имеет язык высокого уровня, Toolbox'ы (оптимизация, Simulink). Поддерживает графику, численные методы, симплекс-метод через linprog, графовые алгоритмы. Широко применяется в инженерии и науке.

**42\. Простейшие вычисления в MATLAB** - работа в командной строке или скриптах. Арифметические операции (+, -, \*, /, ^), переменные (x=5), матрицы (\[1 2;3 4\]), функции (sin, exp). Решение линейных систем A\\b, построение графиков plot(x,y), встроенные константы (pi, i).
